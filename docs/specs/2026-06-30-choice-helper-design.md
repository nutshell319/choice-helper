# 选择助手 — 设计规格说明

> 日期：2026-06-30 | 状态：设计中

## 1. 项目概述

一个本地 HTML 页面，帮助用户在日常和工作场景中做出选择。通过多种趣味交互模式（转盘、抽签、翻牌等），降低"选择困难"带来的心理摩擦。

### 核心目标
- **即开即用**：双击 HTML 文件即可在浏览器中打开，零安装
- **多模式**：覆盖不同心境的决策方式（随机、仪式感、视觉冲击）
- **可复用**：支持保存和加载预设选项列表，避免重复输入

## 2. 技术方案

- **形式**：纯单文件 HTML（CSS/JS 全部内联）
- **存储**：`localStorage` 持久化预设列表和当前选项
- **动画**：CSS Animation + Canvas（转盘绘制）+ requestAnimationFrame
- **依赖**：零外部依赖，不需要服务器或构建工具

## 3. 功能模块

### 3.1 三种核心模式

| 模式 | 描述 | 选项数范围 | 核心动画 |
|------|------|-----------|----------|
| 🎡 幸运转盘 | 彩色扇形转盘，点击旋转后指针停在一个选项上 | 2~20 | Canvas 绘制 + 缓动减速旋转 |
| 🎋 抽签筒 | 签条在筒中翻滚抖动，一支弹出显示结果 | 2~30 | CSS transform + JS 随机运动 |
| 🃏 翻牌/洗牌 | 卡片排列，翻转后洗牌，用户点击翻牌揭晓 | 3~12 | CSS 3D flip + 洗牌动画 |

### 3.2 额外小工具

- **🎲 数字骰子**：输入范围（默认 1~6），滚动动画后给出随机数
- **🪙 抛硬币**：快速二选一，硬币翻转动画

### 3.3 选项输入

- 手动逐条输入：输入框 + 添加按钮，选项以可删除标签展示
- 预设列表管理：
  - 创建/编辑/删除命名列表
  - 一键加载列表到当前输入区
  - JSON 导出/导入备份

### 3.4 数据持久化

- `localStorage` key：`choice-helper-presets`（预设列表）
- `localStorage` key：`choice-helper-current`（当前输入区选项，刷新恢复）
- 清空当前选项、导出全部数据功能

## 4. UI 布局

```
┌──────────────────────────────────────────────┐
│  🎲 选择助手                    [⚙ 管理列表] │  顶部导航
├──────────────────────────────────────────────┤
│  [🎡转盘] [🎋抽签] [🃏翻牌] [🎲骰子] [🪙硬币] │  模式 Tab
├──────────────────────────────────────────────┤
│                                              │
│             当前模式的动画/交互区域             │  主舞台
│             （占据页面主体 ~60% 高度）          │
│                                              │
├──────────────────────────────────────────────┤
│  [输入框___________________] [+添加]          │  输入面板
│  ┌──────┐ ┌──────┐ ┌──────┐                 │
│  │ 选项1 ✕│ │ 选项2 ✕│ │ 选项3 ✕│              │  标签列表
│  └──────┘ └──────┘ └──────┘                 │
│                       [▼ 加载预设列表]        │
└──────────────────────────────────────────────┘
```

- 管理面板为 Modal 弹窗，包含列表的 CRUD 和导入导出
- 响应式设计：桌面端为主要目标，移动端可接受但非重点优化

## 5. 视觉风格

**暖色活力风**：
- 背景：深蓝紫渐变 `#1a1a2e → #16213e`
- 主色/高亮：活力橙金 `#f0a500`
- 强调/按钮：珊瑚粉 `#ff6b6b`
- 转盘扇区：彩虹色系（红橙黄绿青蓝紫循环）
- 卡片：圆角 + 半透明毛玻璃效果 `rgba(255,255,255,0.08)`
- 字体：系统默认中文字体栈（`"Microsoft YaHei", "PingFang SC", sans-serif`）
- 标题可尝试 CSS 渐变文字增加视觉趣味

## 6. 交互细节

- **转盘**：点击"开始"按钮触发旋转，旋转期间禁用按钮，停止后弹出结果提示（Toast/Modal）
- **抽签**：点击"摇一摇"，签筒抖动 2~3 秒后弹出结果
- **翻牌**：洗牌动画约 1.5 秒，之后用户点击任意一张翻牌
- **骰子**：数字快速切换约 1 秒后定格
- **硬币**：3D 翻转约 1.5 秒后落地
- 所有模式在结果揭示后，提供"再来一次"按钮

## 7. 边界情况与错误处理

- 选项数不足（转盘<2、翻牌<3）：禁用开始按钮，显示提示文字
- 选项数超限：显示警告，建议精简
- 选项名为空字符串：不允许添加
- 重复选项：不阻止，但给予视觉提示（可选）
- localStorage 满：静默降级，提示用户导出旧数据
- 预设列表名重复：提示覆盖或重命名
- 导入格式错误：提示 JSON 格式无效

## 8. 非功能需求

- 页面加载时间 < 1 秒
- 动画帧率 ≥ 30fps
- 单文件大小 < 100KB（不含预设数据）
- 支持 Chrome/Firefox/Edge 最新两个大版本
